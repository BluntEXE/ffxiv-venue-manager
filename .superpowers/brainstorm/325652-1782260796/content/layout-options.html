Maintenance Page Layout

Which structure feels right for communicating site status + plugin guidance?

XIV VENUE MANAGER

A — Centred Single Panel

Logo + status + one compact plugin notice. Minimal. On-brand. Fast to read. Best for short outages where users just need "it'll be back soon."

XIV VENUE MANAGER
Scheduled Maintenance
Est. return 04:30 UTC
What's happening
Database updates & server upgrades in progress.
Plugin notice
Clock-in/out, sales & patron logging offline.

B — Two-Column Info Cards

Splits "what's happening" from "plugin impact" into side-by-side cards. More informative. Good when maintenance has a reason worth explaining.

XIV VENUE MANAGER
Under Maintenance
Est. return 04:30 UTC
Plugin behaviour during maintenance
✕Clock-in / Clock-out — will fail with error
✕Patron visit logging — silent fail, visits lost
✕Sale! logging — silent fail, records lost
⚠DTR bar / shift info — may show stale data
✓All features restore automatically on site return

C — Status List

Full per-feature status breakdown with red/yellow/green indicators. Most detailed. Best if you expect users to be mid-event and need to know exactly what works.
